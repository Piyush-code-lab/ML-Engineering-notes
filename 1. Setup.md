# End-to-End Machine Learning Project Setup (Industry Style)

## 1. Objective of This Series

The goal of this series is to build **complete end-to-end machine learning projects** using **industry-level practices**.

This includes:

* Modular coding
* Version control using GitHub
* Project structure
* Environment management
* Packaging ML applications
* Logging & exception handling
* Deployment

### Why This Is Important

In real companies:

* ML projects are **never written in a single notebook**
* Code must be **modular and collaborative**
* Multiple engineers work on the **same codebase**

Therefore, we need:

* Version control
* Standard project structure
* Reproducible environments
* Proper dependency management

These are also **very common topics in data science interviews**.

# 3. Step 1 — Create GitHub Repository

### What to Do

1. Open GitHub
2. Go to **Repositories**
3. Click **New Repository**
4. Name the repository

Example:

```
ml-project
```

Keep it **public** (optional).

Click:

```
Create Repository
```

---

### Why We Create a GitHub Repository

GitHub acts as the **central storage for the project code**.

In real companies:

* Multiple developers work on the same repository
* Code changes are tracked
* Code is merged through branches
* Collaboration becomes easier

Without GitHub:

* Version control becomes chaotic
* Code conflicts become frequent
* Collaboration becomes difficult

# 6. Step 4 — Create Virtual Environment

### Command

```
conda create -p venv python=3.8 -y
```

### Explanation of Parameters

| Parameter      | Meaning                   |
| -------------- | ------------------------- |
| `conda create` | create environment        |
| `-p venv`      | environment name/location |
| `python=3.8`   | specific python version   |
| `-y`           | auto approve installation |

---

### What This Creates

Inside your project folder:

```
ml-project
│
├── venv
```

All installed libraries will go inside this environment.

---

### Why Virtual Environments Are Important

They ensure:

* project isolation
* dependency control
* reproducibility

Example problem without environment:

Project A needs:

```
pandas==1.5
```

Project B needs:

```
pandas==2.0
```

Without environments → conflicts.

With environments → no conflict.

# 17. Step 15 — Create `setup.py`

Create file:

```
setup.py
```

This file converts your project into a **Python package**.

Example structure:

```python
from setuptools import find_packages, setup

setup(
    name="mlproject",
    version="0.0.1",
    author="Krish",
    author_email="krishnaik@gmail.com",
    packages=find_packages(),
    install_requires=["pandas","numpy","seaborn"]
)
```

---

### Why setup.py Is Important

It allows you to:

* build Python packages
* distribute libraries
* install projects easily
* publish to **PyPI**

Example:

```
pip install seaborn
```

Seaborn itself was built using **setup.py**.

# 19. Step 17 — Automating Dependency Reading

Instead of writing dependencies manually inside `setup.py`, we read them from `requirements.txt`.

Example function: Write this function in the setup.py

```python
from typing import List

def get_requirements(file_path:str)->List[str]:
    
    requirements = []

    with open(file_path) as file_obj:
        requirements = file_obj.readlines()

        requirements = [req.replace("\n","") for req in requirements]

    return requirements
```

---

### What This Function Does

1. Reads `requirements.txt`
2. Removes newline characters
3. Returns dependency list

This list is then used inside `setup.py`.
