# 12. Automatic Creation of GitHub Workflow

Once the web app is created, Azure automatically modifies the GitHub repository.

A new folder appears:

```
.github/
```

Inside it:

```
workflows/
```

And a **YAML workflow file** is created.

Example:

```
azure-webapp.yml
```

This file defines the **deployment pipeline**.

---

# 13. Example GitHub Actions Workflow

Example YAML file:

```
name: Build and deploy Python app to Azure Web App

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v2

      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.8'

      - name: Install dependencies
        run: |
          python -m venv antenv
          source antenv/bin/activate
          pip install -r requirements.txt
```

---

# 14. Understanding the GitHub Workflow

---

# 14.1 Trigger

```
on:
  push:
    branches: main
```

This means:

Whenever code is pushed to the **main branch**, the deployment pipeline runs automatically.

---

# 14.2 Runner Environment

```
runs-on: ubuntu-latest
```

GitHub creates a temporary **Ubuntu server** to execute deployment steps.

---

# 14.3 Checkout Repository

```
actions/checkout
```

This step downloads the GitHub repository to the runner environment.

---

# 14.4 Setup Python Environment

```
actions/setup-python
```

Installs the required Python version.

---

# 14.5 Create Virtual Environment

```
python -m venv antenv
```

Creates an isolated Python environment.

This prevents dependency conflicts.

---

# 14.6 Activate Virtual Environment

```
source antenv/bin/activate
```

Activates the environment for installing packages.

---

# 14.7 Install Dependencies

```
pip install -r requirements.txt
```

Installs all required libraries for the ML project.

---

# 14.8 Upload Build Artifacts

The project files are packaged into a **zip file** and uploaded to Azure.

---

# 14.9 Deployment to Web App

The artifacts are deployed to the **Azure Web App environment**.

The application becomes publicly accessible.

---
